# 融資系システム ランディングゾーン

> 融資審査・実行・管理を担うシステムのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | 融資系システム |
| 主な機能 | 融資審査（スコアリング）、融資実行、期中管理、担保管理、返済管理 |
| FISC外部性 | 機微性を有する（個人信用情報を取扱う） |
| 重要度 | **Tier 2（高）** |
| 処理特性 | 審査処理（AI/ML活用含む）+ OLTP + バッチ |
| 可用性要件 | 99.95%以上 |

## Azureサービス構成

| コンポーネント | Azureサービス | 構成 |
|-------------|-------------|------|
| 融資Webアプリ | Azure App Service / AKS | 可用性ゾーン |
| 審査エンジン | Azure Machine Learning | マネージドエンドポイント |
| 融資台帳DB | Azure SQL MI General Purpose | Auto-FO Group |

| 外部信用情報照会 | Azure API Management | CIC/JICC等との接続 |

## セキュリティ特記事項（機微情報保護）

融資系は個人信用情報（要配慮個人情報を含む）を取り扱うため、FISC基準上「機微性を有するシステム」として高い安全対策が求められます。

| 対策 | 実装 | FISC基準 |
|------|------|---------|
| 個人情報暗号化 | Always Encrypted + CMK | 実3 |
| データマスキング | Azure SQL Dynamic Data Masking | 実3 |
| アクセス制御 | 列レベルセキュリティ + 行レベルセキュリティ | 実25 |
| データ分類 | Microsoft Purview 機密度ラベル | 統7 |
| 監査ログ | Azure SQL Auditing + 不変ストレージ | 実10 |
| DLP | Microsoft Purview DLP | 実3 |

## AI/ML活用時の考慮事項（実150〜153）

融資審査にAI/MLを活用する場合、FISC第13版のAI安全対策基準に準拠が必要です。

| 観点 | 対策 |
|------|------|
| 公平性・バイアス | Azure ML の Responsible AI ダッシュボードで公平性指標を監視 |
| 説明可能性 | モデル解釈機能（SHAP/LIME）による審査理由の説明 |
| モデルドリフト | Azure ML モニタリングによるモデル精度の継続監視 |
| 人的関与 | AI判定結果の人間によるレビュー・最終承認プロセス |
