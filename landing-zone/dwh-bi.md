# 情報系・DWH/BI システム ランディングゾーン

> データウェアハウス・ビジネスインテリジェンス・データ分析基盤のAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | 情報系・DWH/BIシステム |
| 主な機能 | 経営情報分析、リスク分析、収益管理、規制報告、マーケティング分析 |
| FISC外部性 | 通常は外部性なし（内部利用） |
| 重要度 | **Tier 3（中）** |
| 処理特性 | バッチETL、OLAP、ダッシュボード、アドホッククエリ |
| 可用性要件 | 99.9%以上 |

## Azureサービス構成

| コンポーネント | Azureサービス | 構成 |
|-------------|-------------|------|
| データ取込み | Azure Data Factory | マネージドVNet統合 |
| データレイク | Azure Data Lake Storage Gen2 | ZRS/GRS + Private Endpoint |
| SQL分析 | Azure Synapse Analytics | Dedicated SQL Pool / Serverless |
| 高度分析・ML | Azure Databricks (Premium) | VNet Injection |
| リアルタイム分析 | Azure Data Explorer | 可用性ゾーン |
| 可視化 | Power BI Premium | 容量ベースライセンス |
| データガバナンス | Microsoft Purview | データカタログ・分類・リネージ |

## セキュリティ特記事項

情報系は複数業務系からデータを集約するため、データガバナンスが特に重要です。

| 対策 | 実装 | FISC基準 |
|------|------|---------|
| データ分類 | Microsoft Purview 機密度ラベル | 統7 |
| 行レベルセキュリティ | Azure Synapse RLS / Power BI RLS | 実25 |
| データマスキング | Dynamic Data Masking | 実3 |
| データリネージ | Microsoft Purview リネージ | 統7 |
| アクセス監査 | Azure Monitor + Synapse 監査ログ | 実10 |
| 個人情報保護 | 匿名化・仮名化処理（Data Factory変換） | 実3 |

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 4時間 |
| **RPO** | < 1時間 |
| 構成 | Single Region + Azure Backup (GRS) |
| 特記 | 規制報告用データは不変ストレージで法定保存期間保持 |
