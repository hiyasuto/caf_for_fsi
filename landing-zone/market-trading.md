# 市場系・トレーディングシステム ランディングゾーン

> 有価証券売買・デリバティブ取引・外国為替取引を担うシステムのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | 市場系・トレーディングシステム |
| 主な機能 | 有価証券売買、デリバティブ取引、外国為替取引、ポジション管理、損益計算 |
| FISC外部性 | 重大な外部性を有する可能性（証券会社の主要業務） |
| 重要度 | **Tier 1〜2（金融機関種別により異なる）** |
| 処理特性 | **超低レイテンシ**（ミリ秒単位の応答要求）、大量同時処理 |
| 可用性要件 | 99.99%以上（取引時間中） |

## Azureサービス構成

| コンポーネント | Azureサービス | 構成 | 理由 |
|-------------|-------------|------|------|
| トレーディングエンジン | AKS (Proximity Placement Group) | 低レイテンシ最適化 | ミリ秒応答要件 |
| リスク計算（VaR等） | Azure Batch / HPC | GPU/HPC対応VM | モンテカルロシミュレーション等 |
| ポジションDB | Azure Cosmos DB (Strong Consistency) | マルチリージョン | リアルタイムポジション管理 |
| 約定DB | Azure SQL MI Business Critical | 可用性ゾーン | ACID保証 |
| マーケットデータ | Azure Event Hubs (Dedicated) | 高スループット | 大量ティックデータの処理 |
| 時系列データ | Azure Data Explorer | リアルタイム分析 | マーケットデータ分析 |
| 対外接続 | Azure ExpressRoute | 専用線 | 取引所・情報ベンダー |

## パフォーマンス設計

市場系は金融システム中で最も厳しいパフォーマンス要件を持ちます。

| 要件 | Azure設計 |
|------|----------|
| レイテンシ < 10ms | Proximity Placement Group + Premium SSD v2 |
| スループット（注文処理） | AKS HPA + KEDA によるイベント駆動スケーリング |
| リスク計算バッチ | Azure Batch + HBv4/NCv5 VM による大規模並列計算 |
| マーケットデータ | Event Hubs Dedicated（数百万イベント/秒） |

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 5分（取引時間中） |
| **RPO** | 0（約定データは損失不可） |
| 構成 | Active-Active（可用性ゾーン3ゾーン） |
| リージョンDR | 西日本 Warm Standby |
| 特記 | 取引所接続の切替手順を含むコンティンジェンシープラン |
