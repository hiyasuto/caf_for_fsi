# ATM系システム ランディングゾーン

> ATM端末管理・ATM取引処理を担うシステムのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | ATM系システム |
| 主な機能 | 現金入出金、振込・送金、残高照会、通帳記帳、提携ATM連携 |
| 設置形態 | 銀行内、コンビニ、駅・空港、ショッピングセンター、移動店舗車 |
| FISC外部性 | 各金融機関の判断による |
| 重要度 | **Tier 2〜3** |
| 処理特性 | リアルタイム取引（OLTP）、24時間運用 |
| 可用性要件 | 99.95%以上 |

## FISC基準上の位置づけ

ATM関連はFISC基準で最も多く言及されるシステムの一つです（287回出現）。FISC第13版では設置形態の多様化（コンビニ、駅、移動店舗車等）に対応した基準見直しが行われています。

**適用される主な基準**:
- 実107〜実110: カード管理・取引監視
- 実119〜実121: ATMコーナー運用管理・防犯
- 設113〜設138: ATM設備基準（物理セキュリティ）

## Azureサービス構成

| コンポーネント | Azureサービス | 構成 |
|-------------|-------------|------|
| ATMスイッチ | AKS (Private) | 可用性ゾーン x3、低レイテンシ |
| 取引DB | Azure SQL MI Business Critical | Auto-FO Group |
| 不正検知 | Microsoft Sentinel + Stream Analytics | リアルタイム異常取引検知 |
| ATM監視 | Azure Monitor + IoT Hub | 端末稼働状態の集中監視 |
| 提携ネットワーク | Azure ExpressRoute | CAFIS、統合ATM接続 |
| HSM | Azure Payment HSM | PIN暗号化・MAC生成 |

## セキュリティ特記事項

| FISC基準 | 要件 | Azure実装 |
|---------|------|----------|
| 実107 | カードの管理方法 | Azure Key Vault（カード鍵管理） |
| 実108 | カード取引犯罪の注意喚起 | — (業務運用) |
| 実110 | カード取引監視 | Microsoft Sentinel（不正パターン検知） |
| 実119 | ATMコーナー運用管理・不正払戻防止 | Azure Payment HSM（PIN検証） |
| 実121 | ATMコーナー防犯体制 | Azure IoT Hub（監視カメラ連携） |

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 15分 |
| **RPO** | 0（取引データ損失不可） |
| 構成 | 可用性ゾーン x3 + リージョンDR |
| 特記 | ATM端末のフォールバック動作（オフライン取引制限） |
