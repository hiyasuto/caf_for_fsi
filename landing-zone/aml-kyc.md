# AML/KYC系システム ランディングゾーン

> マネーロンダリング対策・本人確認を担うシステムのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | AML/KYCシステム（マネロン対策・本人確認） |
| 主な機能 | 取引モニタリング、疑わしい取引の検知・届出、本人確認（eKYC）、制裁リスト照合 |
| FISC外部性 | 機微性を有する（個人情報・取引情報を大量に処理） |
| 重要度 | **Tier 2（高）**（規制要件） |
| 処理特性 | リアルタイム検知 + バッチ分析 + AI/ML |
| 可用性要件 | 99.95%以上 |

## Azureサービス構成

| コンポーネント | Azureサービス | 理由 |
|-------------|-------------|------|
| リアルタイム検知 | Azure Stream Analytics + Event Hubs | 取引ストリームのリアルタイム分析 |
| ルールエンジン | Microsoft Sentinel | 疑わしい取引パターンのカスタム検出ルール |
| eKYC | Azure AI Document Intelligence + Face API | 身分証OCR・顔照合 |
| 制裁リスト照合 | Azure AI Search + Azure SQL DB | ファジーマッチング検索 |
| 不正検知ML | Azure Machine Learning | 異常検知・ネットワーク分析モデル |
| グラフ分析 | Azure Cosmos DB (Gremlin API) | 資金フロー・関係者ネットワーク分析 |
| データガバナンス | Microsoft Purview | 個人情報分類・データリネージ |

## セキュリティ特記事項

AML/KYCシステムは個人情報を大量に処理するため、FISC基準上「機微性を有するシステム」です。

| 対策 | 実装 | FISC基準 |
|------|------|---------|
| 個人情報暗号化 | Always Encrypted + CMK | 実3 |
| 本人確認画像保護 | Azure Blob Storage（CMK + 不変ストレージ） | 実3 |
| データアクセス制御 | RBAC + 行レベルセキュリティ | 実25 |
| 監査ログ | 全照会・検知結果の不変ログ保存 | 実10 |
| AI公平性 | Azure ML Responsible AI ダッシュボード | 実150 |
| データ保持 | 法定保存期間（7年等）の自動管理 | 統7 |

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 1時間（リアルタイム検知）/ < 4時間（バッチ） |
| **RPO** | < 15分 |
| 構成 | 可用性ゾーン + Azure Backup (GRS) |
| 特記 | 取引モニタリングの停止は規制違反リスクがあるため高可用性が必要 |
