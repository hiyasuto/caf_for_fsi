# 対外接続系システム ランディングゾーン

> 全銀ネット・日銀ネット・SWIFT等との接続を担うシステムのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | 対外接続系システム（外部ネットワーク接続基盤） |
| 主な機能 | 全銀ネット中継、日銀ネット接続、SWIFT接続、CAFIS接続、証券取引所接続 |
| FISC外部性 | **重大な外部性を有する** — 金融インフラへの直接接続 |
| 重要度 | **Tier 1（最高）** |
| 処理特性 | 電文中継（リアルタイム）、ファイル転送（バッチ） |
| 可用性要件 | 99.99%以上 |

## FISC基準上の位置づけ

対外接続系は「外部接続」に関するFISC実務基準（実34）および統制基準の外部統制（統20〜統24）が直接適用されます。金融インフラへの接続点として最高レベルのセキュリティと可用性が要求されます。

## 接続先別の構成

| 接続先 | 接続方式 | 暗号化 | 冗長化 |
|-------|---------|-------|-------|
| 全銀ネット | ExpressRoute Private Peering | 回線暗号化 + アプリ層暗号 | 二重化必須 |
| 日銀ネット（BOJ-NET） | ExpressRoute Private Peering | 専用暗号装置 | 二重化必須 |
| SWIFT | SWIFTNet（Alliance Gateway） | SWIFT CSP準拠 | Active-Active |
| CAFIS | ExpressRoute Private Peering | 回線暗号化 | 二重化 |
| でんさいネット | ExpressRoute | 回線暗号化 | 二重化 |
| 証券取引所 | ExpressRoute | 回線暗号化 | 二重化 |

## セキュリティ特記事項

| 対策 | 実装 | FISC基準 |
|------|------|---------|
| ネットワーク分離 | 専用Spoke VNet + NSG厳格制御 | 実14, 実15 |
| 電文暗号化 | Key Vault Managed HSM + 専用暗号装置 | 実4, 実13 |
| 改ざん検知 | MAC（メッセージ認証コード）生成・検証 | 実5 |
| 監査ログ | 全電文の不変ストレージ保存（10年以上） | 実10 |
| アクセス制御 | Entra PIM + 二人制オペレーション | 実25, 実36 |
| 経済安全保障 | 特定重要設備の届出対応 | 統1（経済安全保障推進法） |

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 5分（自動FO） |
| **RPO** | 0（電文損失不可） |
| 構成 | Active-Active（可用性ゾーン3ゾーン） + リージョンDR |
| 回線冗長 | ExpressRoute二重化 + VPNバックアップ |
| 特記 | 対外接続先との切替手順を含むコンティンジェンシープラン |
