# モバイルバンキング ランディングゾーン

> スマートフォンアプリ向けバンキングサービスのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | モバイルバンキング（MB） |
| 主な機能 | 残高照会、振込・送金、QRコード決済、通知、生体認証ログイン |
| FISC外部性 | 各金融機関の判断による |
| 重要度 | **Tier 2〜3** |
| 処理特性 | REST API / GraphQL、プッシュ通知、24/365運用 |
| 可用性要件 | 99.95%以上 |

## Azureサービス構成

| コンポーネント | Azureサービス | 理由 |
|-------------|-------------|------|
| API Gateway | Azure API Management (Premium) | バージョニング・レート制限・認証 |
| バックエンドAPI | Azure Container Apps / AKS | コンテナベース・Autoscale |
| 認証基盤 | Microsoft Entra External ID | FIDO2・生体認証・SMS OTP対応 |
| プッシュ通知 | Azure Notification Hubs | iOS/Android対応・大量配信 |
| セッション | Azure Cache for Redis | トークン管理 |
| DB | Azure SQL Database | 取引履歴・設定情報 |
| CDN | Azure Front Door | 静的アセット配信 |

## モバイル固有のセキュリティ

| 対策 | 実装 | FISC基準 |
|------|------|---------|
| 生体認証 | FIDO2 / WebAuthn via Entra External ID | 実1, 実8 |
| デバイスバインディング | Conditional Access（デバイス準拠チェック） | 実9 |
| 証明書ピニング | アプリ側実装 + Azure Front Door証明書 | 実4 |
| ルート/JB検出 | アプリ側実装（Intune MAM連携可能） | 実6 |
| APIレート制限 | Azure API Management スロットリング | 実14 |
| トークン管理 | OAuth 2.0 + PKCE + リフレッシュトークン | 実9 |
