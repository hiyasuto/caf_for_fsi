# 為替・決済系システム ランディングゾーン

> 内国為替・外国為替・資金決済を担うシステムのAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | 為替・決済系システム |
| 主な機能 | 内国為替（振込・送金）、外国為替、資金決済、口座振替 |
| FISC外部性 | **重大な外部性を有する** — 金融システム全体への波及リスク |
| 重要度 | **Tier 1（最高）** |
| 処理特性 | リアルタイム処理 + バッチ処理（仕向・被仕向） |
| 可用性要件 | 99.99%以上 |
| 対外接続 | 全銀ネット、日銀ネット（BOJ-NET）、SWIFT |

## FISC基準上の位置づけ

為替システムは「重大な外部性を有するシステム」の代表例としてFISC基準に明示されています。障害発生時に他の金融機関やその顧客に広く影響を及ぼし、社会全体に経済的損失を与える可能性があります。

## Azureサービス構成

### コンピューティング

| コンポーネント | Azureサービス | 構成 | 理由 |
|-------------|-------------|------|------|
| 為替処理エンジン | AKS (Private) | 可用性ゾーン x3 | リアルタイム処理・スケーラビリティ |
| バッチ集中処理 | Azure Batch | 専用プール | 全銀ファイル処理 |
| メッセージ変換 | Azure Functions (Premium) | VNet統合 | 電文フォーマット変換 |

### データベース・メッセージング

| コンポーネント | Azureサービス | 構成 | 理由 |
|-------------|-------------|------|------|
| 為替台帳DB | Azure SQL MI Business Critical | Auto-FO Group | ACID保証・高可用性 |
| メッセージキュー | Azure Service Bus Premium | トランザクションセッション | 電文の順序保証・重複排除 |
| 決済用HSM | Azure Payment HSM | Thales payShield 10K | PCI PTS HSM v3準拠 |

### 対外接続

| 接続先 | 接続方式 | Azureサービス |
|-------|---------|-------------|
| 全銀ネット | 専用線（ExpressRoute） | Azure ExpressRoute + NVA |
| SWIFT | SWIFTNet + Alliance Lite2 | Azure 上の SWIFT CSP対応環境 |
| 日銀ネット | 専用線 | ExpressRoute Private Peering |
| CAFIS | 専用線 | ExpressRoute |

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 5分（自動FO）/ < 30分（リージョンDR） |
| **RPO** | 0（同期レプリケーション） |
| データ整合性 | 分散トランザクション + Saga パターン |
| 冪等性 | 全電文にユニーク ID を付与し重複処理を防止 |
| バックアップ | 取引データの不変ストレージ保存（法定保存期間対応） |

## セキュリティ特記事項

| 対策 | 実装 | FISC基準 |
