# AI・生成AI基盤 ランディングゾーン

> 金融機関におけるAI/生成AI活用基盤のAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | AI・生成AI基盤 |
| 主な機能 | 社内文書検索、顧客対応支援、融資審査支援、レポート生成、コード生成支援 |
| FISC外部性 | 用途により異なる（社内利用は外部性なし、顧客向けは要判断） |
| 重要度 | **Tier 3〜4**（用途による） |
| 処理特性 | 推論API（リアルタイム）、ファインチューニング（バッチ） |
| 可用性要件 | 99.9%以上 |
| 適用FISC基準 | **実150〜実153（第13版新設）** |

## FISC AI安全対策基準の要件

| 基準 | 要件 | 設計への反映 |
|------|------|------------|
| 実150 | 方針策定・態勢整備 | AI利用ポリシーの策定、責任者の指定 |
| 実151 | 運用管理方法 | モデル管理・監視・ログ管理の実装 |
| 実152 | 安全対策 | 入出力フィルタリング・データ保護・バイアス対策 |
| 実153 | 教育・注意喚起 | 利用者への研修・ガイドライン周知 |

## Azureサービス構成

| コンポーネント | Azureサービス | 構成 |
|-------------|-------------|------|
| 生成AI | Azure OpenAI Service | Private Endpoint、日本リージョン |
| RAG検索 | Azure AI Search (Standard) | ベクトル検索 + セマンティックランカー |
| コンテンツ安全 | Azure AI Content Safety | 入出力フィルタリング、Prompt Shields |
| API管理 | Azure API Management | レート制限、認証、利用量追跡 |
| ML基盤 | Azure Machine Learning | モデル管理、Responsible AI |
| 特徴量基盤 | Azure Databricks | 特徴量ストア、モデル学習 |
| データガバナンス | Microsoft Purview | 学習データの分類・リネージ |
| プロジェクト管理 | Azure AI Foundry | 評価・テスト・デプロイ管理 |

## セキュリティ設計（実152対応）

### 多層ガードレール

```
利用者 → API Management → Content Safety → Azure OpenAI → Content Safety → 利用者
           │                 │                               │
           ├─ 認証(Entra ID)  ├─ 入力フィルタ                   ├─ 出力フィルタ
           ├─ レート制限       ├─ Prompt Shields                ├─ 著作権フィルタ
           ├─ 利用量管理       ├─ PII検出                       ├─ PII検出
           └─ 監査ログ        └─ ジェイルブレイク検出             └─ 根拠チェック
```

| リスク | 対策 | Azure実装 |
|-------|------|----------|
| 機密データ入力 | DLP・PII検出 | Content Safety + Purview DLP |
| プロンプトインジェクション | 入力検証 | Content Safety Prompt Shields |
| ハルシネーション | RAG（検索拡張生成） | Azure AI Search + グラウンディング |
| バイアス | 公平性評価 | Azure ML Responsible AI ダッシュボード |
| 著作権侵害 | 著作権フィルタ | Azure OpenAI 著作権フィルター |
| データプライバシー | データ所在地管理 | 日本リージョン、入力は学習非使用 |
| 不正利用 | 利用量監視 | API Management + Azure Monitor |

## Azure OpenAI のデータプライバシー

| 項目 | Azure OpenAI の保証 |
|------|-------------------|
| 入力データの学習利用 | **使用されない** |
| データ所在地 | 日本リージョン指定可能 |
| 不正利用フィルタ | 30日以内に削除（オプトアウト申請可能） |
| 暗号化 | 保存時・転送時暗号化 |
| アクセス制御 | Azure RBAC + Private Endpoint |
