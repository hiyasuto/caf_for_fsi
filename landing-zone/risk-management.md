# リスク管理系システム ランディングゾーン

> 市場リスク・信用リスク・オペレーショナルリスク管理のAzure設計ガイダンス

## システム概要

| 項目 | 内容 |
|------|------|
| システム名 | リスク管理系システム |
| 主な機能 | 市場リスク（VaR）、信用リスク、オペリスク、ALM、自己資本比率計算、ストレステスト |
| FISC外部性 | 通常は外部性なし（内部管理） |
| 重要度 | **Tier 2〜3**（規制報告の期限遵守が必要） |
| 処理特性 | **大規模バッチ計算**（モンテカルロシミュレーション等）、日次/月次レポート |
| 可用性要件 | 99.9%以上 |

## Azureサービス構成

| コンポーネント | Azureサービス | 構成 | 理由 |
|-------------|-------------|------|------|
| VaR計算 | Azure Batch (HPC VM) | HBv4/NCv5 VM | モンテカルロシミュレーション（数千シナリオ） |
| ストレステスト | Azure Batch + AKS | オンデマンドスケール | 四半期ストレステスト時のバースト計算 |
| 市場データ格納 | Azure Data Lake Storage Gen2 | ZRS | 大量時系列データ |
| 分析エンジン | Azure Synapse Analytics / Databricks | 専用プール | 大量データの集計・分析 |
| 結果DB | Azure SQL MI | General Purpose | リスク指標・規制報告データ |
| レポート | Power BI Premium | 専用容量 | 規制報告・経営レポート |

## パフォーマンス設計

リスク計算は金融機関で最も計算リソースを消費するワークロードの一つです。

| 計算種別 | Azure構成 | 計算規模（例） |
|---------|----------|-------------|
| 市場リスク（VaR） | Azure Batch — HBv4 VM x 100台 | 10,000シナリオ × ポートフォリオ |
| 信用リスク | Azure Batch — Dv5 VM x 50台 | デフォルト確率計算 |
| ストレステスト | AKS + KEDA — バーストスケール | 規制シナリオ × 全ポートフォリオ |
| xVA計算 | Azure Batch — NCv5 (GPU) | ニューラルネットワーク近似 |

**コスト最適化**: Azure Spot VM を計算ノードの一部に活用（計算結果のチェックポイント機能付き）

## 可用性・DR設計

| 要件 | 設計 |
|------|------|
| **RTO** | < 4時間 |
| **RPO** | < 1時間 |
| 構成 | Single Region（計算リソースはオンデマンド） |
| 特記 | 規制報告期限に間に合うよう、計算リソースの予約計画が重要 |
