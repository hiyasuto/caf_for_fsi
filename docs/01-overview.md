# 01 — フレームワーク概要

## 目的と背景

本フレームワークは、金融機関等がMicrosoft Azureクラウド上にシステムを構築・運用する際に、FISC安全対策基準・解説書（第13版）への準拠を実現するための統合的なガイダンスです。

Azure Cloud Adoption Framework（CAF）のクラウド導入方法論と Well-Architected Framework（WAF）のアーキテクチャ設計原則を活用し、FISC基準の要件をクラウドネイティブな形で実現する方法を示します。

## FISCフレームワークの構造

FISC安全対策基準は以下の4つの基準から構成されます：

### 統制基準（36項目）
経営層・管理者が果たすべきITガバナンス・ITマネジメントに関する基準。方針策定、組織体制、外部委託管理、クラウドサービス利用に関する統制を含む。

| 区分 | 内容 | 基準番号 |
|-----|------|---------|
| 内部の統制 | 方針・計画 | 統1〜統3 |
| 内部の統制 | 組織体制 | 統4〜統19 |
| 外部の統制 | 外部委託 | 統20〜統23 |
| 外部の統制 | クラウドサービス | 統24 |
| 外部の統制 | FinTech等連携 | 統25〜統27 |
| 外部の統制 | サプライチェーン | 統28 |

### 実務基準（152項目）
システムの開発・運用における実務的な安全対策基準。

| 区分 | 内容 | 基準番号 |
|-----|------|---------|
| 技術的安全対策 | データ保護・認証・不正プログラム対策 | 実1〜実22 |
| 運用管理 | マニュアル整備・アクセス管理・鍵管理 | 実23〜実34 |
| 運用管理 | システム運用 | 実36〜実62 |
| 障害・災害対策 | 復旧・コンティンジェンシー・バックアップサイト | 実71〜実74 |
| 開発・変更 | 開発プロセス・品質 | 実75〜実101 |
| 電子決済 | カード・ATM・インターネット | 実107〜実121 |
| AI安全対策 | AI利用方針・運用 | 実150〜実153 |

### 設備基準（134項目）
データセンター・営業店等の物理的安全対策基準。

### 監査基準（2項目）
システム監査およびサイバーセキュリティ監査に関する基準。

## FISC基準の適用アプローチ

### リスクベースアプローチ

FISC第13版では、リスクベースアプローチの考え方を採用しています：

1. **情報システムのリスク特性に応じた安全対策目標の設定**
2. **経営資源全体を視野に入れた資源配分の決定**
3. **重大な外部性・機微性を有するシステムへの高い安全対策の適用**

### 基準分類

各基準項目は以下のように分類されます：

- **基礎基準**: すべての金融機関等に適用が必要
- **付加基準**: リスク特性に応じて適用を判断

### 適用区分

| 区分 | 説明 |
|-----|------|
| 共通 | 建物・チャネルに依存せず適用 |
| データセンター | データセンターに適用 |
| 本部・営業店等 | 本部・営業店に適用 |
| 提携チャネル | 流通・小売店舗等 |
| ダイレクトチャネル | インターネット・モバイル等 |

## Azure CAF との統合

Azure Cloud Adoption Framework の各フェーズにFISC要件をマッピングします：

```
FISC要件                    Azure CAF フェーズ
─────────────────────────────────────────────
統1-3 方針・計画        →   戦略 (Strategy) / 計画 (Plan)
統4-19 組織体制         →   ガバナンス (Govern)
統20-28 外部委託・クラウド →  準備 (Ready) / ガバナンス (Govern)
実1-19 技術的安全対策    →   準備 (Ready) / 導入 (Adopt)
実25-62 運用管理        →   管理 (Manage)
実71-73 障害・災害対策   →   管理 (Manage)
実75-101 開発・変更     →   導入 (Adopt)
監1 監査               →   ガバナンス (Govern)
```

## Azure WAF 5つの柱との対応

| WAF の柱 | FISC対応領域 | 主な基準 |
|---------|-------------|---------|
| **信頼性 (Reliability)** | 障害対策・災害対策・バックアップ | 実39〜45, 実71〜74, 設1〜設18 |
| **セキュリティ (Security)** | 認証・暗号化・アクセス制御・サイバー対策 | 実1-19, 実25-30, 統4-5, 統1-1〜統5-5 |
| **コスト最適化 (Cost Optimization)** | 経営資源の適切な配分 | 統2, 統3 |
| **オペレーショナルエクセレンス (Operational Excellence)** | 運用管理・変更管理・監視 | 実34-62, 実75-101 |
| **パフォーマンス効率 (Performance Efficiency)** | 負荷管理・キャパシティ管理 | 実101 |

## 第13版の新設・主要改訂基準

### サイバーセキュリティ強化（新設）
- **統1-1**: サイバーセキュリティ対策に関する基本方針の整備
- **統1-2**: サイバーセキュリティ対策に関する規程等及び業務プロセスの整備
- **統4-1**: サイバーセキュリティ管理のための経営資源及び人材計画
- **統4-2**: サイバーセキュリティ管理態勢の監視及び牽制
- **統5-1**: 情報資産の適切な管理
- **統5-2**: サイバーセキュリティリスクの特定・評価及びリスク対応計画
- **統5-3**: 脆弱性管理手続きの策定
- **統5-4**: サイバーセキュリティに関する演習・訓練
- **統5-5**: サイバーセキュリティに係る教育・研修
- **統28**: サプライチェーンを考慮したサイバーセキュリティリスク管理
- **実14-1**: サイバー攻撃の端緒検知のための監視・分析
- **実14-2**: 脆弱性診断及びペネトレーションテスト
- **実73-1**: サイバー攻撃を想定したインシデント対応計画及びコンティンジェンシープラン
- **監1-1**: サイバーセキュリティを対象とした内部監査

### AI安全対策（新設）
- **実150**: AIの利用に係る方針の策定と態勢の整備
- **実151**: AIの適切な運用管理方法の策定
- **実152**: AIに係る安全対策
- **実153**: AIの利用に係る教育、注意喚起等
